# Page Type Notes

Adjust the critique to the page type.

## Landing page or marketing site

Prioritize:
- first-screen brand hit
- narrative flow
- product promise clarity
- supporting proof and call to action
- whether the page earns its visual drama

Watch for:
- boxed hero with weak presence
- too many sections with no story arc
- stock saas card grids replacing product storytelling

## Product page or feature page

Prioritize:
- feature explanation
- screenshots or product framing
- relationship between value proposition and detail
- whether the design helps the product feel credible and differentiated

## Web app or internal tool

Prioritize:
- scan speed
- navigation clarity
- state visibility
- density balance
- whether beauty is helping or hurting productivity

Watch for:
- pretty but inefficient layouts
- large decorative chrome around routine tasks
- weak distinction between primary and secondary actions

## Dashboard or analytics ui

Prioritize:
- task hierarchy
- comparison clarity
- information grouping
- chart versus table balance
- what needs to be seen first versus explored later

Watch for:
- identical tile sizes for unequal information
- decorative widgets with low analytical value
- over-fragmented grids that slow reading

## Redesign request

Decide first whether the issue is mainly:
- style
- structure
- content strategy
- interaction logic
- implementation quality

Do not prescribe a visual makeover when the real issue is information architecture.
