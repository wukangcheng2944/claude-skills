# Golden Cases

Use a small fixed canary set in every iteration so live noise does not dominate the optimization loop.

## Case 1: Corning / optical fiber chain

Purpose:
- test whether the agent can turn a US optical-fiber move into a China supplier chain instead of generic AI concept chatter
- test proactive logic and stage judgment

Failure modes:
- only explains why Corning moved
- maps directly to generic AI names without industry chain logic
- treats any pullback as thesis broken

## Case 2: SanDisk / storage narrative

Purpose:
- test whether the agent can map a concrete product / storage catalyst into relevant A-share chains and concept spillover
- check whether it separates supply-chain logic from broad “AI hardware” concept noise

## Case 3: Google TPU -> PCB / server chain

Purpose:
- test multi-hop reasoning without hallucinating certainty
- force confidence tagging and relationship description

Failure modes:
- overstates certainty on indirect links
- skips provenance and confidence

## Case 4: Loser signal / risk alert

Purpose:
- ensure bearish cases are framed as holding-risk alerts, not short recommendations

Failure modes:
- writes “short” framing
- assigns too much conviction to weak downside concept transmission

## Case 5: Mainstream-media diffusion

Purpose:
- test whether the agent recognizes that once media and big accounts widely discuss a theme, the edge may already be thinner

Failure modes:
- treats crowded narratives as fresh alpha
- cannot distinguish first reaction from late-stage consensus
